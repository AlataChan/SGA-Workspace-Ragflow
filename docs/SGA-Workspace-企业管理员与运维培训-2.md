## 2. 课件 A：管理员操作手册

> 目标：让管理员能独立完成"从零配置系统 → 用户可以正常使用"的全流程。

---

### A-M0 初始化与首次登录（3–5 页）

#### ADM-01 系统初始化（只做一次）

**本页目标**：学员知道初始化是什么、什么时候做、做完了就不用再做了。

**页内要点**：
- 系统第一次部署后，需要创建第一个管理员账号
- 入口：浏览器访问 `http://<地址>/setup`
- 初始化只能做一次，之后这个页面会自动失效
- 初始化后跳转到登录页，用刚创建的账号登录

**演示步骤**：
1. 打开浏览器，输入地址，看到初始化页面
2. 填写：用户名（登录用）、员工号、手机号、邮箱、显示姓名、职位、密码
3. 点击"初始化"按钮
4. 跳转到登录页，用刚填的账号密码登录
5. 成功进入 `/admin/company`

**注意事项（讲师口述）**：
> 用户名和密码一定要记住！如果忘记密码，需要联系运维通过数据库重置，没有"忘记密码"自助功能。建议用一个好记但安全的密码，并安全保存。

**练习与验收**：
- 学员能用自己创建的账号成功登录，并看到管理后台菜单

---

#### ADM-02 管理后台导航

**本页目标**：学员能在管理后台里快速找到任何一个功能入口。

**页内要点**：
- 左侧菜单包含所有管理功能
- 菜单项：公司设置 / 主题 / 部门管理 / 组织架构 / 用户管理 / AI 助手 / 知识图谱
- 点击菜单项直接跳转对应页面，URL 会变化
- 右上角显示当前登录的管理员账号

**演示步骤**：
1. 登录后看到左侧导航菜单
2. 逐一点击每个菜单项，说明这个页面是干什么的
3. 展示如何回到上一页（浏览器后退或点菜单）

---

### A-M1 公司设置（3–5 页）

#### ADM-03 设置公司名称与 Logo

**本页目标**：学员能修改公司名称，并成功上传 Logo。

**页内要点**：
- 入口：左侧菜单 → **公司设置**（`/admin/company`）
- 公司名称会显示在系统各处（登录页、顶部导航等）
- Logo 会显示在顶部导航栏，建议使用透明背景的 PNG
- 修改后点击保存，页面会自动刷新以显示最新效果

**演示步骤**：
1. 点击左侧"公司设置"
2. 在"公司名称"输入框里改名 → 点击保存
3. 在"Logo"区域点击上传 → 选择图片文件（JPG/PNG，建议不超过 2MB）
4. 看到预览图 → 点击保存
5. 页面刷新后，验证顶部导航已显示新 Logo

**常见问题**：
- Logo 上传失败 → 检查文件格式（只支持 JPG/PNG）和大小（不超过 2MB）
- 保存后看不到变化 → 强制刷新浏览器（Ctrl+Shift+R / Cmd+Shift+R）

**练习与验收**：
- 学员成功上传 Logo，并在导航栏里看到它

---

### A-M2 主题设置（2–3 页）

#### ADM-04 切换界面主题

**本页目标**：学员能根据场合选择合适的界面颜色主题。

**页内要点**：
- 入口：左侧菜单 → **主题**（`/admin/theme`）
- 三个选项：浅色 / 深色 / 跟随系统
  - **浅色**：白色背景，适合白天使用、会议投屏
  - **深色**：深色背景，护眼，适合长时间使用
  - **跟随系统**：自动跟随电脑设置，推荐给普通用户
- 主题只影响视觉效果，不影响任何数据或权限

**演示步骤**：
1. 点击左侧"主题"
2. 依次点击三个选项，实时看到界面变化
3. 选择"浅色"作为演示环境主题（方便投影）

### A-M3 部门管理（8–12 页）

#### ADM-05 为什么要先建部门

**本页目标**：学员理解部门的作用，以及为什么它是所有配置的基础。

**页内要点**：
- 部门是"分类标签"：让 AI 助手（Agent）按部门分组展示给用户
- 用户在工作台看到的是"XX 部门的助手"，而不是一堆没有归属的助手
- 创建 Agent 时必须选择归属部门，所以**部门要先建好**
- 部门数量建议与公司实际组织架构对应（不用完全一致，以实用为主）

**示例**：
> 假设公司有销售部、客服部、技术部三个部门，就创建三个对应的部门，然后把各自的 AI 助手放进去。用户在工作台就能看到清晰的分组。

---

#### ADM-06 新建部门

**本页目标**：学员能独立创建一个新部门，并配置好所有信息。

**页内要点**：
- 入口：左侧菜单 → **部门管理**（`/admin/departments`）→ 点击"新建部门"
- 必填项：**部门名称**
- 选填项：描述、图标（从预设图标库选择）、排序序号
- 排序序号越小，在工作台里越靠前显示
- 创建后在列表里可以看到：Agent 数量、在线 Agent 数、用户数

**演示步骤**：
1. 点击左侧"部门管理"
2. 点击页面右上角"新建部门"按钮
3. 填写部门名称（例如：销售助手组）
4. 填写描述（例如：销售部门专用 AI 助手）
5. 点击图标选择，挑一个合适的图标
6. 填写排序序号（例如：1 表示排第一）
7. 点击"保存"
8. 在列表里找到刚创建的部门，确认显示正确

**练习与验收**：
- 学员创建 2 个部门，名称、图标各不同，排序不同
- 在列表里能看到两个部门

---

#### ADM-07 编辑与停用部门

**本页目标**：学员能修改部门信息，以及知道"停用"的含义和后果。

**页内要点**：
- 在部门列表里，点击部门右侧的"编辑"按钮可修改信息
- **停用部门**：该部门下的 Agent 不再显示给用户（但数据还在，可以重新启用）
- 停用不等于删除——停用的部门和 Agent 数据完整保留
- 什么时候停用：部门解散、临时下线某组助手时使用

**演示步骤**：
1. 找到刚建的部门，点击"编辑"
2. 修改描述 → 保存
3. 点击"停用"按钮 → 确认操作
4. 切换到工作台（`/workspace`），验证该部门的 Agent 不再显示
5. 回到部门管理 → 重新启用

**注意事项（讲师口述）**：
> 停用部门后，已经有权限的用户刷新工作台就看不到这个部门的助手了。如果用户反馈"助手不见了"，优先检查部门是否被停用。

---

#### ADM-08 查看组织架构树

**本页目标**：学员能用组织架构页快速了解整体人员分布。

**页内要点**：
- 入口：左侧菜单 → **组织架构**（`/admin/organization`）
- 展示所有部门和部门下的用户，树形结构，可展开/收起
- 可以搜索用户名，快速定位某人在哪个部门
- "未分配"分组：显示还没有归属部门的用户（需要管理员手动分配）

**演示步骤**：
1. 点击左侧"组织架构"
2. 展开/收起部门节点
3. 在搜索框输入用户名，看到自动高亮展开路径
4. 找到"未分配"用户，说明下一步要在用户管理里给他们分配部门

### A-M4 用户管理（14–20 页）

#### ADM-09 用户与角色概念

**本页目标**：学员清楚两种角色的区别，以及为什么角色设置很重要。

**页内要点**：
- 系统只有两种角色：**管理员（ADMIN）** 和 **普通用户（USER）**
- 管理员：能进入 `/admin/*` 管理后台，可以配置一切
- 普通用户：只能进入工作台（`/workspace`），使用授权给他们的 AI 助手
- **一个重要规则**：如果把某人设为管理员，他就能修改系统所有配置——请谨慎设置

**类比（讲师口述）**：
> 就像商场的"员工通道"和"顾客通道"。管理员有员工通道门禁，能进后台管理；普通用户只走顾客通道，只能使用前台功能。

---

#### ADM-10 创建用户账号

**本页目标**：学员能独立为新员工创建账号并分配到部门。

**页内要点**：
- 入口：左侧菜单 → **用户管理**（`/admin/users`）→ 点击"新建用户"
- **必填字段**：
  - 用户名（登录时使用，建议用工号或英文名）
  - 员工号（唯一标识，同一公司不能重复）
  - 手机号
  - 显示名称（中文名，工作台里展示）
  - 部门（从已建的部门里选）
  - 初始密码（建议告知用户后让他们自行修改）
- **选填字段**：邮箱、英文名、职位、头像
- 角色默认为"普通用户（USER）"，只有需要时才改为 ADMIN

**演示步骤**：
1. 点击"用户管理" → 点击"新建用户"
2. 填写用户名：`zhangsan`
3. 填写员工号：`EMP001`
4. 填写手机号、显示名称（张三）
5. 选择部门：销售助手组
6. 设置初始密码
7. 点击保存
8. 在用户列表里找到张三，确认信息正确

**练习与验收**：
- 学员创建 2 个用户，分配到不同部门
- 在用户列表里能找到他们，部门显示正确

---

#### ADM-11 编辑用户信息

**本页目标**：学员能修改用户的基本信息和所属部门。

**页内要点**：
- 在用户列表里点击用户的"编辑"按钮
- 可以修改：显示名称、部门、职位、邮箱、头像
- **什么不建议随便改**：用户名（影响登录）、员工号（影响唯一性）
- **转部门**：直接在编辑里改部门即可，该用户对原部门 Agent 的权限**不会自动变化**（需要手动调整授权）

**注意事项（讲师口述）**：
> 员工转岗时，记得两件事：一是在这里改部门，二是去授权管理里撤销旧部门的 Agent 权限、添加新部门的 Agent 权限。光改部门，权限不会自动跟着变。

---

#### ADM-12 停用与启用用户

**本页目标**：学员知道如何处理员工离职或临时封禁的情况。

**页内要点**：
- 在用户列表里，每个用户旁边有"停用"/"启用"开关
- **停用后**：该用户无法登录，已有的 token 会失效
- **停用不等于删除**：数据完整保留，随时可以重新启用
- **员工离职处理流程**：停用账号 → 不需要删除（保留历史记录）

**演示步骤**：
1. 找到刚建的测试用户
2. 点击"停用"
3. 用该用户账号尝试登录（验证确实无法登录）
4. 回到管理后台 → 点击"启用" → 再次验证可以正常登录

---

#### ADM-13 搜索与筛选用户

**本页目标**：学员能在大量用户中快速找到特定的人。

**页内要点**：
- 页面顶部有搜索框，输入姓名/用户名/员工号即可搜索
- 可以按部门筛选：下拉选择部门名称
- 可以按角色筛选：管理员 / 普通用户
- 列表支持分页，每页显示 10/20/50 条（可调整）

**演示步骤**：
1. 在搜索框输入"张"，看到所有名字含"张"的用户
2. 选择部门筛选条件，验证只显示该部门的用户
3. 清空搜索条件，看到所有用户
