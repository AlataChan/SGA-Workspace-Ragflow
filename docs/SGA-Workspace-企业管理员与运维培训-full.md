## 0. 讲师说明

### 0.1 课件结构

- **共用开场（8–10 页）**：系统概览、术语解释、两类角色分工
- **课件 A：管理员操作手册**（建议 2.5–3 小时）：从零配置整个系统
- **课件 B：普通用户操作手册**（建议 0.5–1 小时）：日常使用工作台

### 0.2 每页固定格式

1. **这页学完你能做什么**（可验证的目标）
2. **页内要点**（3–5 条，PPT 展示）
3. **演示步骤**（一步一步点击路径）
4. **学员练习**（动手完成一件事 + 验收标准）

### 0.3 演示环境准备

开课前请准备好：
- 1 个管理员账号（ADMIN 角色）
- 2 个普通用户账号（USER 角色，无任何授权）
- 至少 3 个部门（名称清晰，图标各不同）
- 至少 3 个 Agent（建议覆盖 Dify / RAGFlow 各一个）
- 至少 1 个知识图谱（RAGFlow 已连通）
- 至少 3 份测试文档（PDF / DOCX / TXT 各一份）

## 1. 共用开场（管理员 + 用户都参加）

### COMMON-01 封面

- 课程名称、版本、日期、讲师姓名
- 今天谁来上课：管理员 vs 普通用户，课程目标不一样

### COMMON-02 这个系统是干什么的

**一句话**：SGA Workspace 是企业统一的 AI 助手平台，让员工通过一个入口使用公司部署的所有 AI 助手。

**页内要点**：
- 公司部署多个 AI 助手（Agent），按部门分组
- 员工登录后，只看到自己有权限的 AI 助手
- 管理员负责配置、普通用户负责使用
- 所有对话、知识库、权限都在这一个系统里管理

**类比说明**（讲师口述）：
> 就像公司的"AI 助手门户"。管理员是后台配置人员，负责把 AI 助手上架、分配权限；普通用户是前台使用者，打开工作台就能找到自己的 AI 助手开始对话。

### COMMON-03 两类角色分工

| 角色 | 谁是这个角色 | 他们做什么 |
|------|-------------|-----------|
| 管理员（ADMIN） | IT 负责人、系统管理员 | 创建部门、添加用户、配置 Agent、管理权限 |
| 普通用户（USER） | 全体员工 | 登录系统、选择 AI 助手、发起对话 |

**关键规则**：
- 普通用户看不到管理后台，只有管理员能进入 `/admin/*` 页面
- 用户能用哪个 AI 助手，完全由管理员授权决定

### COMMON-04 名词解释（避免后面讲课混乱）

| 名词 | 简单解释 |
|------|---------|
| Agent（AI 助手） | 一个可以对话的 AI 角色，背后连接 Dify/RAGFlow 等 AI 平台 |
| 部门 | 用于给 Agent 分组展示（如"销售部助手""技术部助手"） |
| 知识图谱 | 企业知识库，AI 助手可以从中查找信息来回答问题 |
| 授权 | 管理员设置"某用户可以用某个 Agent 或知识图谱" |
| 工作台（/workspace） | 普通用户日常使用的主页面 |
| 管理后台（/admin） | 管理员专属的配置页面 |
| 连接测试 | 验证 Agent 背后的 AI 平台是否正常连通 |

### COMMON-05 完整使用流程（从上线到日常）

```
运维部署完成
    ↓
管理员初始化（/setup）→ 创建第一个管理员账号
    ↓
管理员配置公司信息（名称、Logo）
    ↓
管理员创建部门
    ↓
管理员创建 Agent（并测试连接）
    ↓
管理员创建用户账号（分配部门）
    ↓
管理员给用户授权 Agent / 知识图谱
    ↓
用户登录工作台（/workspace）开始使用
```

**讲师重点**：上面每一步都有前后依赖，顺序不能乱。比如：先有部门，才能把 Agent 归到部门；先有 Agent，才能给用户授权。

### COMMON-06 登录入口与页面地图

- 系统入口：`http://<公司域名或 IP>:8100`
- 首次访问：自动跳转到初始化页（`/setup`）
- 之后访问：跳转到登录页（`/auth/login`）
- 管理员登录后进入：`/admin/company`（管理后台）
- 普通用户登录后进入：`/workspace`（工作台）

### A-M0 初始化与首次登录（3–5 页）

#### ADM-01 系统初始化（只做一次）

**本页目标**：学员知道初始化是什么、什么时候做、做完了就不用再做了。

**页内要点**：
- 系统第一次部署后，需要创建第一个管理员账号
- 入口：浏览器访问 `http://<地址>/setup`
- 初始化只能做一次，之后这个页面会自动失效
- 初始化后跳转到登录页，用刚创建的账号登录

**演示步骤**：
1. 打开浏览器，输入地址，看到初始化页面
2. 填写：用户名（登录用）、员工号、手机号、邮箱、显示姓名、职位、密码
3. 点击"初始化"按钮
4. 跳转到登录页，用刚填的账号密码登录
5. 成功进入 `/admin/company`

**注意事项（讲师口述）**：
> 用户名和密码一定要记住！如果忘记密码，需要联系运维通过数据库重置，没有"忘记密码"自助功能。建议用一个好记但安全的密码，并安全保存。

**练习与验收**：
- 学员能用自己创建的账号成功登录，并看到管理后台菜单

### A-M1 公司设置（3–5 页）

#### ADM-03 设置公司名称与 Logo

**本页目标**：学员能修改公司名称，并成功上传 Logo。

**页内要点**：
- 入口：左侧菜单 → **公司设置**（`/admin/company`）
- 公司名称会显示在系统各处（登录页、顶部导航等）
- Logo 会显示在顶部导航栏，建议使用透明背景的 PNG
- 修改后点击保存，页面会自动刷新以显示最新效果

**演示步骤**：
1. 点击左侧"公司设置"
2. 在"公司名称"输入框里改名 → 点击保存

**常见问题**：
- Logo 上传失败 → 检查文件格式（只支持 JPG/PNG）和大小（不超过 2MB）
- 保存后看不到变化 → 强制刷新浏览器（Ctrl+Shift+R / Cmd+Shift+R）

**练习与验收**：
- 学员成功上传 Logo，并在导航栏里看到它

### A-M2 主题设置（2–3 页）

#### ADM-04 切换界面主题

### A-M3 部门管理（8–12 页）

#### ADM-05 为什么要先建部门

**本页目标**：学员理解部门的作用，以及为什么它是所有配置的基础。

**页内要点**：
- 部门是"分类标签"：让 AI 助手（Agent）按部门分组展示给用户
- 用户在工作台看到的是"XX 部门的助手"，而不是一堆没有归属的助手
- 创建 Agent 时必须选择归属部门，所以**部门要先建好**
- 部门数量建议与公司实际组织架构对应（不用完全一致，以实用为主）

**示例**：
> 假设公司有销售部、客服部、技术部三个部门，就创建三个对应的部门，然后把各自的 AI 助手放进去。用户在工作台就能看到清晰的分组。

---

#### ADM-06 新建部门

**本页目标**：学员能独立创建一个新部门，并配置好所有信息。

**页内要点**：
- 入口：左侧菜单 → **部门管理**（`/admin/departments`）→ 点击"新建部门"
- 必填项：**部门名称**
- 选填项：描述、图标（从预设图标库选择）、排序序号
- 排序序号越小，在工作台里越靠前显示
- 创建后在列表里可以看到：Agent 数量、在线 Agent 数、用户数

**演示步骤**：
1. 点击左侧"部门管理"
2. 点击页面右上角"新建部门"按钮
3. 填写部门名称（例如：销售助手组）
4. 填写描述（例如：销售部门专用 AI 助手）
5. 点击图标选择，挑一个合适的图标
6. 填写排序序号（例如：1 表示排第一）
7. 点击"保存"
8. 在列表里找到刚创建的部门，确认显示正确

**练习与验收**：
- 学员创建 2 个部门，名称、图标各不同，排序不同
- 在列表里能看到两个部门

#### ADM-07 编辑与停用部门

**本页目标**：学员能修改部门信息，以及知道"停用"的含义和后果。

**页内要点**：
- 在部门列表里，点击部门右侧的"编辑"按钮可修改信息
- **停用部门**：该部门下的 Agent 不再显示给用户（但数据还在，可以重新启用）
- 停用不等于删除——停用的部门和 Agent 数据完整保留
- 什么时候停用：部门解散、临时下线某组助手时使用

**演示步骤**：
1. 找到刚建的部门，点击"编辑"
2. 修改描述 → 保存
3. 点击"停用"按钮 → 确认操作
4. 切换到工作台（`/workspace`），验证该部门的 Agent 不再显示
5. 回到部门管理 → 重新启用

**注意事项（讲师口述）**：
> 停用部门后，已经有权限的用户刷新工作台就看不到这个部门的助手了。如果用户反馈"助手不见了"，优先检查部门是否被停用。

#### ADM-08 查看组织架构树

**本页目标**：学员能用组织架构页快速了解整体人员分布。

**页内要点**：
- 入口：左侧菜单 → **组织架构**（`/admin/organization`）
- 展示所有部门和部门下的用户，树形结构，可展开/收起
- 可以搜索用户名，快速定位某人在哪个部门
- "未分配"分组：显示还没有归属部门的用户（需要管理员手动分配）

**演示步骤**：
1. 点击左侧"组织架构"
2. 展开/收起部门节点
3. 在搜索框输入用户名，看到自动高亮展开路径
4. 找到"未分配"用户，说明下一步要在用户管理里给他们分配部门

### A-M4 用户管理（14–20 页）

#### ADM-09 用户与角色概念

**本页目标**：学员清楚两种角色的区别，以及为什么角色设置很重要。

**页内要点**：
- 系统只有两种角色：**管理员（ADMIN）** 和 **普通用户（USER）**
- 管理员：能进入 `/admin/*` 管理后台，可以配置一切
- 普通用户：只能进入工作台（`/workspace`），使用授权给他们的 AI 助手
- **一个重要规则**：如果把某人设为管理员，他就能修改系统所有配置——请谨慎设置

**类比（讲师口述）**：
> 就像商场的"员工通道"和"顾客通道"。管理员有员工通道门禁，能进后台管理；普通用户只走顾客通道，只能使用前台功能。

#### ADM-10 创建用户账号

**本页目标**：学员能独立为新员工创建账号并分配到部门。

**页内要点**：
- 入口：左侧菜单 → **用户管理**（`/admin/users`）→ 点击"新建用户"
- **必填字段**：

  - 员工号（唯一标识，同一公司不能重复）
  - 手机号
  - 显示名称（中文名，工作台里展示）
  - 部门（从已建的部门里选）
  - 初始密码（建议告知用户后让他们自行修改）
- **选填字段**：邮箱、英文名、职位、头像
- 角色默认为"普通用户（USER）"，只有需要时才改为 ADMIN

**演示步骤**：
1. 点击"用户管理" → 点击"新建用户"
2. 填写用户名：`zhangsan`
3. 填写员工号：`EMP001`
4. 填写手机号、显示名称（张三）
5. 选择部门：销售助手组
6. 设置初始密码
7. 点击保存
8. 在用户列表里找到张三，确认信息正确

**练习与验收**：
- 学员创建 2 个用户，分配到不同部门
- 在用户列表里能找到他们，部门显示正确

---

#### ADM-11 编辑用户信息

**本页目标**：学员能修改用户的基本信息和所属部门。

**页内要点**：
- 在用户列表里点击用户的"编辑"按钮
- 可以修改：显示名称、部门、职位、邮箱、头像
- **什么不建议随便改**：用户名（影响登录）、员工号（影响唯一性）
- **转部门**：直接在编辑里改部门即可，该用户对原部门 Agent 的权限**不会自动变化**（需要手动调整授权）

**注意事项（讲师口述）**：
> 员工转岗时，记得两件事：一是在这里改部门，二是去授权管理里撤销旧部门的 Agent 权限、添加新部门的 Agent 权限。光改部门，权限不会自动跟着变。

---

#### ADM-12 停用与启用用户

**本页目标**：学员知道如何处理员工离职或临时封禁的情况。

**页内要点**：
- 在用户列表里，每个用户旁边有"停用"/"启用"开关
- **停用后**：该用户无法登录，已有的 token 会失效
- **停用不等于删除**：数据完整保留，随时可以重新启用
- **员工离职处理流程**：停用账号 → 不需要删除（保留历史记录）

**演示步骤**：
1. 找到刚建的测试用户
2. 点击"停用"
3. 用该用户账号尝试登录（验证确实无法登录）
4. 回到管理后台 → 点击"启用" → 再次验证可以正常登录

---

#### ADM-13 搜索与筛选用户

**本页目标**：学员能在大量用户中快速找到特定的人。

**页内要点**：
- 页面顶部有搜索框，输入姓名/用户名/员工号即可搜索
- 可以按部门筛选：下拉选择部门名称
- 可以按角色筛选：管理员 / 普通用户
- 列表支持分页，每页显示 10/20/50 条（可调整）

**演示步骤**：
1. 在搜索框输入"张"，看到所有名字含"张"的用户
2. 选择部门筛选条件，验证只显示该部门的用户
3. 清空搜索条件，看到所有用户

### A-M5 AI 助手（Agent）管理（18–24 页）

#### ADM-14 什么是 Agent，如何理解它

**本页目标**：学员能用自己的话解释 Agent 是什么，知道创建 Agent 需要什么前提。

**页内要点**：
- **Agent = AI 助手**：一个有名字、有头像、有专属职责的 AI 角色
- 每个 Agent 背后连接一个 AI 平台（Dify、RAGFlow 等），用平台来提供对话能力
- 用户看到的 Agent 有名字（如"销售分析助手"）、有头像、归属某个部门
- 创建 Agent 前必须已经：① 有归属部门 ② 有 AI 平台的 API 地址和密钥

**支持的 AI 平台**：

| 平台名称 | 适用场景 |
|---------|---------|
| Dify | 通用对话、知识库问答、工作流 |
| RAGFlow | 基于文档的问答（知识检索增强） |
| OpenAI | GPT 系列模型 |
| Claude | Anthropic Claude 系列模型 |
| 自定义 | 公司自建的 AI 服务 |

---

#### ADM-15 创建 Agent —— 基本信息

**本页目标**：学员能填写 Agent 的基本信息（名称、头像、部门等）。

**页内要点**：
- 入口：左侧菜单 → **AI 助手**（`/admin/agents`）→ 点击"新建助手"
- **必填字段**：
  - 中文名称（用户看到的名字，例如：销售分析助手）
  - 归属部门（从已建部门中选择）
  - 平台类型（选 Dify / RAGFlow / OpenAI 等）
- **选填字段**：
  - 英文名称
  - 职位/岗位描述（例如：负责分析销售数据）
  - 头像图片（建议上传，让用户更容易辨认）
  - 展示照片（工作台里的大图）
  - 排序序号（同部门内的显示顺序）

**演示步骤**：
1. 点击"AI 助手" → 点击"新建助手"
2. 填写中文名称：`销售分析助手`
3. 选择部门：销售助手组
4. 填写描述：帮助分析销售数据、生成报表
5. 上传头像图片
6. 填写排序序号：1

#### ADM-16 创建 Agent —— 平台配置（Dify 示例）

**本页目标**：学员能配置 Dify 平台的 Agent，并成功测试连接。

**页内要点**：
- 选择平台"Dify"后，出现 Dify 专属配置字段：
  - **API 地址**（Base URL）：Dify 部署的服务器地址，例如 `https://dify.company.com`
  - **API 密钥**（API Key）：在 Dify 管理后台生成的密钥
  - **超时时间**：等待 AI 回复的最长秒数（建议默认即可）
- 填完后必须点击"测试连接"，确认 Agent 可以正常工作
- 测试成功显示绿色"连接正常"；失败会显示错误原因

**演示步骤**：
1. 在平台类型选择"Dify"
2. 填写 API 地址（演示用准备好的 Dify 地址）
3. 填写 API 密钥
4. 点击"测试连接"
5. 看到"连接成功"提示
6. 点击"保存"

**连接失败时怎么排查**：

| 错误现象 | 可能原因 | 处理方式 |
|---------|---------|---------|
| 网络超时 | Dify 服务地址不对或网络不通 | 确认 API 地址是否正确 |
| 401 未授权 | API 密钥错误 | 重新从 Dify 后台复制密钥 |
| 404 找不到 | App ID 错误（如有此字段） | 检查 Dify 应用 ID |
| 连接被拒绝 | 防火墙或网络隔离 | 联系运维开通网络访问 |

### A-M6 授权管理（用户能用什么由这里决定）（14–20 页）

#### ADM-20 授权的核心逻辑

**本页目标**：学员彻底理解"为什么用户看不到助手，授权是根本原因"。

**页内要点**：
- **规则**：用户默认什么 Agent 都看不到，必须由管理员显式授权
- **两种可授权的资源**：
  1. AI 助手（Agent）
  2. 知识图谱（Knowledge Graph）
- 授权是"点名制"：只有被授权的资源，用户才能在工作台看到
- 工作台 = 用户被授权资源的汇总展示

**用一句话记住**：
> "没授权 = 用户什么都看不到"

**排查优先级（讲师重点讲）**：

用户反馈"找不到某个助手"时，第一件事检查的就是授权，不是看 Agent 是否在线：
1. 有没有授权给这个用户？（最常见原因）
2. Agent 是否被停用？
3. Agent 是否离线？

#### ADM-21 给用户授权 Agent

**本页目标**：学员能为指定用户添加 Agent 使用权限。

- 授权操作：点击"添加"→ 搜索用户名 → 选中 → 确认
- 授权后：用户刷新工作台即可看到新增的助手（无需重新登录）

**演示步骤**（从用户侧入口）：
1. 进入"用户管理"，点击张三的用户名进入详情
2. 找到页面中的"AI 助手权限"板块

### A-M7 知识图谱管理（10–14 页）

#### ADM-25 知识图谱是什么

**本页目标**：学员用自己的话能解释知识图谱的作用。

**页内要点**：
- 知识图谱 = 公司的"AI 知识库"：把公司文档、规范、产品资料等变成 AI 能理解和检索的结构化数据
- 用户对着 AI 助手提问时，AI 可以从知识图谱里查到相关内容再回答——答案更准、更基于公司实际情况
- 知识图谱基于 RAGFlow 平台（由运维部署），管理员在这里配置连接信息
- 知识图谱需要单独授权给用户（跟 Agent 授权分开）

**类比（讲师口述）**：
> 把知识图谱想象成"公司内部百科全书"。没有它，AI 只能靠通用知识回答问题；有了它，AI 能翻到公司自己的资料再回答，准确度大幅提升。

---

#### ADM-26 创建知识图谱配置

**本页目标**：学员能创建一个知识图谱并测试连接。

**页内要点**：
- 入口：左侧菜单 → **知识图谱**（`/admin/knowledge-graphs`）→ 点击"新建"
- **必填字段**：
  - 名称（例如：产品手册知识库）
  - 描述（例如：包含所有产品规格和使用说明）
  - RAGFlow 服务地址（API URL）
  - RAGFlow API 密钥
  - 知识库 ID（KB ID，在 RAGFlow 中查找）
- 填完后点击"测试连接"确认可以访问
- 测试成功后点击保存

**演示步骤**：
1. 点击"知识图谱" → 点击"新建"
2. 填写名称和描述
3. 填写 RAGFlow 服务地址（演示用已准备好的地址）
4. 填写 API 密钥和 KB ID
5. 点击"测试连接" → 看到成功提示
6. 保存

**知识图谱的关键指标**（建好后可以查看）：
- **节点数**：知识库里有多少个知识实体
- **关系数**：知识实体之间有多少连接
- **最后同步时间**：上次从 RAGFlow 同步数据的时间
- **最后错误**：如果同步失败，这里显示原因

#### ADM-27 授权与验证

**本页目标**：学员能给用户授权知识图谱，并在用户侧验证效果。

**页内要点**：
- 配置好知识图谱 ≠ 用户能用：还需要授权
- 授权方式：用户详情 → 知识图谱权限 → 添加（同 Agent 授权流程）
- 用户被授权后，在工作台侧边栏可以看到并使用知识图谱

**演示步骤**：
1. 给张三授权"产品手册知识库"
2. 张三登录工作台
3. 验证张三能看到知识图谱，并能在对话中使用它

| 切换深色/浅色主题 | 右上角用户图标 → 设置 → 主题 |

#### USER-09 遇到问题时怎么办
