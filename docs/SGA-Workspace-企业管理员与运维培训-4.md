### A-M6 授权管理（用户能用什么由这里决定）（14–20 页）

#### ADM-20 授权的核心逻辑

**本页目标**：学员彻底理解"为什么用户看不到助手，授权是根本原因"。

**页内要点**：
- **规则**：用户默认什么 Agent 都看不到，必须由管理员显式授权
- **两种可授权的资源**：
  1. AI 助手（Agent）
  2. 知识图谱（Knowledge Graph）
- 授权是"点名制"：只有被授权的资源，用户才能在工作台看到
- 工作台 = 用户被授权资源的汇总展示

**用一句话记住**：
> "没授权 = 用户什么都看不到"

**排查优先级（讲师重点讲）**：

用户反馈"找不到某个助手"时，第一件事检查的就是授权，不是看 Agent 是否在线：
1. 有没有授权给这个用户？（最常见原因）
2. Agent 是否被停用？
3. Agent 是否离线？

---

#### ADM-21 给用户授权 Agent

**本页目标**：学员能为指定用户添加 Agent 使用权限。

**页内要点**：
- 入口：左侧菜单 → **用户管理** → 找到用户 → 点击用户名进入详情 → 找到"AI 助手权限"区域
- 或者：左侧菜单 → **AI 助手** → 找到 Agent → 进入详情 → 找到"授权用户"区域
- 授权操作：点击"添加"→ 搜索用户名 → 选中 → 确认
- 授权后：用户刷新工作台即可看到新增的助手（无需重新登录）

**演示步骤**（从用户侧入口）：
1. 进入"用户管理"，点击张三的用户名进入详情
2. 找到页面中的"AI 助手权限"板块
3. 点击"添加助手"
4. 在弹窗里搜索并选择"销售分析助手"
5. 点击确认
6. 让张三（另一台电脑或隐身窗口登录）刷新工作台
7. 验证张三能看到"销售分析助手"

**练习与验收**：
- 学员给 1 个用户授权 2 个不同 Agent
- 用该用户账号登录工作台，确认 2 个 Agent 都出现了

---

#### ADM-22 撤销 Agent 授权

**本页目标**：学员能收回用户对 Agent 的使用权限。

**页内要点**：
- 在用户详情 → AI 助手权限列表里，找到要撤销的 Agent → 点击"移除"
- 撤销后：用户刷新工作台，该 Agent 立即消失
- 撤销不影响历史对话记录（已有对话还能查看，但不能发新消息）

---

#### ADM-23 给用户授权知识图谱

**本页目标**：学员能为用户添加知识图谱访问权限。

**页内要点**：
- 操作方式与授权 Agent 完全相同，只是对象变成了"知识图谱"
- 入口：用户详情 → 找到"知识图谱权限"板块 → 点击"添加"
- 授权后：用户在工作台可以看到并使用该知识图谱

---

#### ADM-24 按部门批量授权（效率技巧）

**本页目标**：学员知道如何一次性给整个部门的用户授权同一个 Agent。

**页内要点**：
- 入口：左侧菜单 → **AI 助手** → 找到 Agent → 进入详情 → "批量授权"功能
- 可以选择一个部门，将该 Agent 授权给部门下所有用户
- 可以选择是否包含子部门用户
- 适用场景：新上线一个 Agent，需要立即授权给整个团队

**演示步骤**：
1. 进入某个 Agent 的详情页
2. 找到"批量授权"按钮
3. 在弹窗里选择部门：销售助手组
4. 勾选"包含子部门"（可选）
5. 点击确认
6. 验证该部门下所有用户都能在工作台看到这个 Agent

### A-M7 知识图谱管理（10–14 页）

#### ADM-25 知识图谱是什么

**本页目标**：学员用自己的话能解释知识图谱的作用。

**页内要点**：
- 知识图谱 = 公司的"AI 知识库"：把公司文档、规范、产品资料等变成 AI 能理解和检索的结构化数据
- 用户对着 AI 助手提问时，AI 可以从知识图谱里查到相关内容再回答——答案更准、更基于公司实际情况
- 知识图谱基于 RAGFlow 平台（由运维部署），管理员在这里配置连接信息
- 知识图谱需要单独授权给用户（跟 Agent 授权分开）

**类比（讲师口述）**：
> 把知识图谱想象成"公司内部百科全书"。没有它，AI 只能靠通用知识回答问题；有了它，AI 能翻到公司自己的资料再回答，准确度大幅提升。

---

#### ADM-26 创建知识图谱配置

**本页目标**：学员能创建一个知识图谱并测试连接。

**页内要点**：
- 入口：左侧菜单 → **知识图谱**（`/admin/knowledge-graphs`）→ 点击"新建"
- **必填字段**：
  - 名称（例如：产品手册知识库）
  - 描述（例如：包含所有产品规格和使用说明）
  - RAGFlow 服务地址（API URL）
  - RAGFlow API 密钥
  - 知识库 ID（KB ID，在 RAGFlow 中查找）
- 填完后点击"测试连接"确认可以访问
- 测试成功后点击保存

**演示步骤**：
1. 点击"知识图谱" → 点击"新建"
2. 填写名称和描述
3. 填写 RAGFlow 服务地址（演示用已准备好的地址）
4. 填写 API 密钥和 KB ID
5. 点击"测试连接" → 看到成功提示
6. 保存

**知识图谱的关键指标**（建好后可以查看）：
- **节点数**：知识库里有多少个知识实体
- **关系数**：知识实体之间有多少连接
- **最后同步时间**：上次从 RAGFlow 同步数据的时间
- **最后错误**：如果同步失败，这里显示原因

#### ADM-27 授权与验证

**本页目标**：学员能给用户授权知识图谱，并在用户侧验证效果。

**页内要点**：
- 配置好知识图谱 ≠ 用户能用：还需要授权
- 授权方式：用户详情 → 知识图谱权限 → 添加（同 Agent 授权流程）
- 用户被授权后，在工作台侧边栏可以看到并使用知识图谱

**演示步骤**：
1. 给张三授权"产品手册知识库"
2. 张三登录工作台
3. 验证张三能看到知识图谱，并能在对话中使用它
